设置padding-top、margin-top: 百分比; 这里的百分比是相对于父级的width。
下面说的“实际宽度”：指盒子实际占用的宽度，包括 content \ padding \ border，也即是document.getElementById('xx').offsetWidth

标准模型(content-box):

设置width就是设置content的宽度

实际宽度 = width + padding * 2 + border * 2
IE模型(border-box):

设置width就是设置实际宽度

实际宽度 = width

content宽度 = width - padding * 2 - border * 2
边距重叠: 父元素没有设置margin-top，而子元素设置了margin-top,然后设置的这个边距居然作用到父元素去了。

看这里，box3sub设置的margin-top作用到其父元素box3上去了。
解决方案：
1.设置子元素float属性不为none（脱离文档流）
2.设置父元素为绝对定位/* position: absolute; or fiexd */
3.设置父元素display为inline-block,table-cell,table-caption,flex,inine-flex
4.设置父元素overflow不为visible
5.假如其父元素是根元素就没有这个问题
